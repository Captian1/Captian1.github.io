#Android架构

标签分类（空格隔开）：Android

Android系统是一个软件组件的栈，其架构包含以下五大组件：

  • 应用程序/Applications
  • 应用程序框架/Application Framework
  • 系统库与安卓运行时/Native C/C++ Libraries Android Runtime
  • 硬件抽象层/Hardware abstraction layer(HAL)
  • Lniux内核/Linux kernel

这五大组件的基本介绍如下所述：

应用程序层

顶层中拥有Android所有的应用程序，例如浏览器、王者荣耀、微信、支付宝以及高德地图等等，还包括我们自己编写的应用。

应用程序框架

应用框架层是Android开发的基础，这也是Android开发人员打交道最多的层。它以Java类的形式为应用程序提供许多高级服务，比如活动管理器、窗口管理器、内容提供者、视图系统、包管理器、电话管理器、资源管理器、位置管理器、通知管理器和XMPP服务等服务。

系统库与安卓运行时

系统库

这一层通过一些C/C++库来为Android系统提供了主要的特性支持。如SQLite库提供了数据库的支持，OpenGL|ES库提供了3D绘图的支持，Webkit库提供了浏览器内核的支持等。

安卓运行时

同样在这一层还有Android运行时库，它主要提供了一些核心库，能够允许开发者使用Java语言来编写Android应用。另外Android运行时库中还包含了Dalvik虚拟机，它使得每一个Android应用都能运行在独立的进程当中，并且拥有一个自己的Dalvik虚拟机实例。相较于Java虚拟机，Dalvik是专门为移动设备定制的，它针对手机内存、CPU性能有限等情况做了优化处理。

硬件抽象层

硬件抽象层（HAL）可定义一个标准接口以供硬件商实现。借助HAL可以顺利实现相关功能，不会影响更加高级的系统。HAL实现会被封装成模块，并且由Android系统适时地加载。

Lunix内核

Android核心系统服务依赖于Linux3.6内核，如安全性、内存管理、进程管理、网络协议栈和驱动模型。Linux内核也是作为硬件与软件栈的抽象层。同时，内核处理所有Linux优势工作，比如网络和大量的驱动，这些驱动包括显示驱动、摄像头驱动、键盘驱动、WiFi驱动、Audio驱动、flash内存驱动、Binder（IPC）驱动、电源管理等。
